# Weekly Site Audit Checklist

Use this checklist each time a new week goes live. Copy the checklist into a new notes file (e.g. `notes-scratch-week2.md`) and fill it in.

## Course: _______________  Week: ___

### Schedule Page

- [ ] Schedule page loads at the correct URL
- [ ] Title and heading are correct
- [ ] All day rows are present with correct dates, topics, and summaries
- [ ] All day links in the Topic column resolve (no 404s)
- [ ] Graded assignments alert box is present and lists only current week's assignments
- [ ] No stale assignments from previous weeks
- [ ] Unit summary is accurate (if applicable)
- [ ] Next week preview is accurate (if applicable)

### Day Pages

- [ ] Day 1 — loads, correct title/date, objectives/warmup/work session/closing render
- [ ] Day 2 — loads, correct title/date, all sections render
- [ ] Day 3 — loads, correct title/date, all sections render
- [ ] Day 4 — loads, correct title/date, all sections render
- [ ] Day 5 — loads, correct title/date, all sections render

### Content Accuracy

- [ ] H1 date headings match frontmatter dates
- [ ] Day references are accurate (e.g. "yesterday" / "earlier this week" point to the right day)
- [ ] External links work (Clever, Scratch, Code.org, Office 365, forms, etc.)
- [ ] Warmup instructions match the intended activity
- [ ] Closing/next steps accurately preview the following day
- [ ] No leftover content from a previous version of the lesson

### Frontmatter Consistency

- [ ] All days have matching frontmatter fields (title, date, description, day_number, weight, units, tags, resources, draft, toc)
- [ ] `draft: false` on all published days
- [ ] Descriptions are accurate and not stale

### Sidebar & Navigation

- [ ] Sidebar lists all days in correct order under the week heading
- [ ] Breadcrumbs are correct on each page
- [ ] Previous/next week navigation works (if applicable)

### Cleanup

- [ ] No stale pages from renamed/moved content (spot-check old URLs return 404)
- [ ] CTLS HTML calendar matches the site schedule
- [ ] `todo.md` updated with any new prep items for the week
